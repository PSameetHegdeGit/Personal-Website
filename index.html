Projects

LSM Storage Engine

I implemented a log-structured merge tree based storage engine. A memtable for in memory storage was used. I provided three options for memtable representation: AVL tree, red black tree, and binary search tree. After a memtable size limit was reached, the memtable was flushed into a segment file stored on disk. Periodically, segment files would be compacted and merged to remove duplicate data entries, which thereby reduced disk memory usage. Additionally a write ahead log was implemented to backup in-memory memtable in the case of a crash.

Java

Collision Detection and Resolution

We randomly generated two dimensional prisms over a grid. We then used the GJK algorithm, based on taking the Minkowski difference and triple cross product of prisms, to find intersections between them. For those prisms that have an intersection, we found the minimal vector needed to separate intersecting prisms. This was based on the EPA algorithm. We completed this project in order to understand fundamental mechanics of collision detection and separation in video games.

Modeling Hardy Weinberg

This Unity game simulates the frequency of specific genotypes in a population over generations. In this world, we make several assumptions of the environment as well as the population such that the population's genotypic frequencies in some later generation stays consistent. This consistency is the Hardy Weinberg equilibrium. The Unity game is coded with c# and communicates with a Flask backend. This Flask backend takes several parameters from the game and uses these values to determine a random genotypic frequency and the changes in this genotypic frequency over time (these frequencies eventually reach the Hardy Weinberg Equilibrium).

More information on the Hardy Weinberg equilibrium can be found here: https://www.nature.com/scitable/definition/hardy-weinberg-equilibrium-122/

Unity | C# | Flask
